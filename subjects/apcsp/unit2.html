AP Computer Science Principles -- Unit 2: The Internet

What Is the Internet?

The internet is a global network of interconnected computers that communicate using agreed-upon rules called protocols. No single entity owns or controls it.

The internet was designed to be fault tolerant -- if one path is blocked, data reroutes around it. This decentralized design was intentional: the original ARPANET was built to survive partial failures (including nuclear attack).
  • The internet connects billions of devices through a mixture of physical cables, fiber optics, cellular networks, and satellite links
  • The World Wide Web (WWW) is a service that runs ON the internet -- a collection of linked documents accessed via browsers. The internet also carries email, file transfers, streaming, and more.
  • Bandwidth: the maximum amount of data that can be transmitted per second (measured in Mbps, Gbps)
  • Latency: the time delay for data to travel from source to destination

Packets and Packet Switching

Data sent over the internet is broken into small chunks called packets. Each packet travels independently and may take a different route to the destination. They are reassembled at the destination.

Packet ComponentContains
HeaderSource IP, destination IP, packet number, total packet count, protocol info
Body (payload)The actual chunk of data being transmitted

Why packet switching?

  • Efficient: network capacity is shared; no dedicated line needed per connection
  • Fault tolerant: if one router fails, packets reroute automatically
  • Scalable: works whether the network has 10 or 10 billion connected devices

IP Addresses and DNS

Every device on the internet has a unique IP address. Humans use domain names (google.com) instead; the Domain Name System (DNS) translates names to IP addresses.

VersionFormatExampleTotal Addresses
IPv44 numbers 0-255, dot-separated192.168.1.1About 4.3 billion
IPv68 groups of 4 hex digits, colon-separated2001:0db8:85a3::8a2e:0370:7334About 3.4 x 10^38

IPv4 addresses are nearly exhausted; IPv6 was designed to provide essentially unlimited addresses for all devices worldwide.

DNS lookup process:

  1. You type "wikipedia.org" in your browser
  2. Your computer asks a DNS resolver (usually your ISP's)
  3. The resolver queries root nameservers, then TLD servers (.org), then Wikipedia's nameserver
  4. The IP address is returned and your browser connects to it
  5. The result is cached to speed up future lookups

Protocols -- TCP/IP and HTTP

Protocols are standardized rules that let different devices and software communicate reliably. The internet runs on a layered stack of protocols.

ProtocolLayerPurpose
HTTP / HTTPSApplicationTransfer web pages; HTTPS adds encryption (TLS)
TCPTransportReliable, ordered delivery; confirms receipt; retransmits lost packets
UDPTransportFast, no confirmation; used for video streaming, gaming, DNS
IPInternetAddresses and routes packets between networks
TCP vs. UDP trade-off: TCP guarantees delivery (good for files, web pages) but is slower. UDP is faster but some packets may be lost -- acceptable for video calls where a dropped frame is better than a delay.

Routing and Redundancy

When you send data, routers read the destination IP in each packet and forward it toward the destination. Each router only needs to know its neighbors -- not the full path.

  • The internet's routing structure is decentralized -- there is no single routing authority
  • Redundancy: multiple paths exist between any two points, so the network continues functioning even if paths fail
  • Scalability: the network can grow by adding nodes without redesigning the whole system
  • traceroute: a tool that shows every router (hop) a packet passes through on its way to a destination

The World Wide Web

The Web is built on three core technologies:

TechnologyRole
HTMLStructure and content of web pages
CSSStyling and layout
JavaScriptInteractivity and dynamic behavior

HTTP request cycle:

  1. Browser sends a GET request to the server's IP address on port 80 (HTTP) or 443 (HTTPS)
  2. Server responds with an HTML document and a status code (200 OK, 404 Not Found, 500 Server Error, etc.)
  3. Browser parses the HTML and makes additional requests for CSS, images, and scripts
  4. Browser renders the complete page

Net Neutrality and Open Standards

The internet's power comes partly from open standards -- protocols that anyone can implement without paying licensing fees. This allows competition and innovation at every layer.

  • Net neutrality: the principle that ISPs must treat all internet traffic equally, regardless of source, destination, or content type. Without it, ISPs could throttle or block specific services.
  • Open protocols: HTTP, TCP/IP, DNS are all open standards maintained by organizations like IETF and W3C
  • The web's success is largely due to Tim Berners-Lee placing the original web protocols into the public domain
