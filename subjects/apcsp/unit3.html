AP Computer Science Principles -- Unit 3: Programming

What Is a Program?

A program is a set of instructions that a computer executes to accomplish a task. Programs are written in a programming language that must eventually be translated to binary machine code that the CPU can execute.

The AP CSP exam uses a generic pseudocode called AP Pseudocode. You are not required to know any specific programming language, but you must understand the pseudocode conventions used on the exam.
ConceptAP Pseudocode Syntax
Assignmenta <-- 5
Display outputDISPLAY(a)

If statementIF (condition) { ... }
If-elseIF (condition) { ... } ELSE { ... }
Repeat N timesREPEAT n TIMES { ... }
Repeat untilREPEAT UNTIL (condition) { ... }
List (array)myList <-- [1, 2, 3]
For each in listFOR EACH item IN myList { ... }

Variables and Data Types

A variable stores a value that can change as the program runs. Variables have a data type that determines what kind of value they can hold.

Conditionals

Conditionals let a program make decisions: execute certain code only if a condition is true.

IF (score >= 90) { DISPLAY("A") } ELSE IF (score >= 80) { DISPLAY("B") } ELSE { DISPLAY("Below B") }

Loops (Iteration)

Loops repeat a block of code. The two main types are count-controlled and condition-controlled.

// Count-controlled: runs exactly 5 times REPEAT 5 TIMES { DISPLAY("Hello") } // Condition-controlled: runs until condition is true count <-- 1 REPEAT UNTIL (count > 10) { DISPLAY(count) count <-- count + 1 } // For each: iterates over a list scores <-- [85, 92, 78] FOR EACH s IN scores { DISPLAY(s) }
An infinite loop occurs when a REPEAT UNTIL condition never becomes true. The program will run forever (or until forcibly stopped). Always make sure your loop condition can eventually be met.

Procedures (Functions)

A procedure (function) is a named block of code that can be called by name. Procedures promote reuse and make programs easier to read and maintain.

// Define a procedure PROCEDURE greet(name) { DISPLAY("Hello, " + name) } // Call it greet("Alice") greet("Bob") // Procedure that returns a value PROCEDURE square(x) { RETURN x * x } result <-- square(5) // result is now 25
ConceptMeaning
ParameterA variable in the procedure definition that receives a value when called
ArgumentThe actual value passed to the procedure at call time
Return valueA value the procedure sends back to the caller
AbstractionHiding implementation details behind a named interface

Lists and Traversal

Lists store multiple values in a single variable. AP CSP lists are 1-indexed (first element is at index 1).

names <-- ["Alice", "Bob", "Carol"] // Access by index DISPLAY(names[1]) // Alice DISPLAY(names[3]) // Carol // Append, insert, remove APPEND(names, "Dave") INSERT(names, 2, "Eve") REMOVE(names, 1) // Find length len <-- LENGTH(names)
AP pseudocode is 1-indexed (names[1] is the first element). Most real programming languages like Python and JavaScript are 0-indexed (names[0] is the first element). Do not confuse the two on the exam.

Debugging and Testing

Finding and fixing errors is an essential part of programming.

Error TypeDescriptionExample
Syntax errorViolates the rules of the language; program won't runMissing brace, misspelled keyword
Runtime errorProgram crashes during executionDividing by zero, accessing invalid list index
Logic errorProgram runs but produces wrong outputOff-by-one in a loop, wrong formula

Testing strategies:

  • Test with normal inputs, edge cases (minimum/maximum), and invalid inputs
  • Trace through code by hand (desk checking) to verify logic
  • Use display/print statements to inspect variable values at key points
  • Test individual procedures before combining them
