AP Computer Science Principles -- Unit 1: Digital Information

How Computers Represent Data

All data in a computer -- numbers, text, images, audio, video -- is ultimately stored as binary: sequences of 0s and 1s. A single 0 or 1 is called a bit. 8 bits = 1 byte.

Computers use binary because electronic circuits have two reliable states: on (1) and off (0). Trying to distinguish more than two voltage levels introduces errors.

Converting binary to decimal: Write out the place values (128, 64, 32, 16, 8, 4, 2, 1), multiply each digit by its place value, and sum.

Representing Text -- ASCII and Unicode

Text is represented by mapping characters to numbers, then storing those numbers in binary.

StandardBitsCharacters SupportedNotes
ASCII7 bits (128 values)English letters, digits, basic symbolsSimple; too small for global use
Extended ASCII8 bits (256 values)ASCII + accented charactersNot standardized across systems
Unicode (UTF-8)Variable (1-4 bytes)All scripts worldwide, emojiUniversal standard; backward compatible with ASCII
The letter 'A' is ASCII 65 = binary 01000001. Every character you type is ultimately a number the computer looks up in a table.

Representing Images -- Pixels and Color

Images are stored as a grid of pixels. Each pixel stores a color value.

  • Grayscale: 1 byte per pixel; values 0 (black) to 255 (white)
  • RGB color: 3 bytes per pixel -- one each for Red, Green, Blue (0-255 each)
  • Image resolution: width x height in pixels. A 1920x1080 image has 2,073,600 pixels = about 6 MB uncompressed
  • Metadata: Images can embed additional data like GPS location, camera model, and timestamp

Compression

Files can be made smaller through compression. The two types behave very differently:

TypeHow It WorksRestorationUse Cases
LosslessRemoves redundancy; encodes patterns more efficientlyPerfect reconstruction of originalZIP, PNG, FLAC; any data where exact accuracy matters
LossyDiscards data the human eye/ear is unlikely to noticeOriginal cannot be fully recoveredJPEG, MP3, MP4; good enough quality at much smaller size

Run-length encoding (lossless example): Instead of storing AAAAABBB, store 5A3B. Only useful when data has long repeated runs.

For the AP exam: Lossless = no data lost, can fully restore. Lossy = some data permanently discarded, cannot restore -- but results in much smaller files. This is a trade-off: file size vs. quality/accuracy.

Data Overflow and Limits

Because computers use a fixed number of bits to store numbers, there are limits to what values they can represent. When a value exceeds the maximum, overflow occurs.

  • An 8-bit unsigned integer can hold 0 to 255 (2^8 - 1). Adding 1 to 255 overflows back to 0.
  • Floating point numbers have limited precision -- very large or very small values introduce rounding errors
  • Overflow bugs can cause real-world failures (the Ariane 5 rocket crash was caused by an integer overflow)
