AP Computer Science Principles -- Unit 6: Cybersecurity

Why Cybersecurity Matters

As more of society depends on networked systems, the consequences of security failures grow: financial theft, healthcare disruptions, election interference, espionage, and infrastructure attacks.

No system is perfectly secure. Security is always a trade-off: stronger security usually means more inconvenience, cost, or reduced functionality. The goal is to make attacks difficult enough that the attacker's cost exceeds their benefit.
  • The CIA triad: the three goals of information security
  • Confidentiality: only authorized parties can read the data
  • Integrity: data has not been altered without authorization
  • Availability: systems and data are accessible when needed

Encryption

Encryption transforms readable data (plaintext) into unreadable form (ciphertext) using a key. Only someone with the correct key can decrypt it.

TypeHow It WorksUse CasesKey Feature
Symmetric encryptionSame key encrypts and decryptsEncrypting stored files; AESFast; but how do you safely share the key?
Asymmetric encryptionPublic key encrypts; private key decrypts (or vice versa for signatures)HTTPS, SSH, digital signatures; RSASolves the key-sharing problem; slower

How HTTPS works (simplified):

  1. Your browser requests a site over HTTPS
  2. The server sends its public key and a digital certificate
  3. Your browser verifies the certificate is signed by a trusted Certificate Authority (CA)
  4. Browser uses the server's public key to securely exchange a shared session key
  5. All further communication is encrypted using that session key (symmetric, for speed)
A digital certificate is like a government-issued ID for a website. It proves the website is who it claims to be. If a certificate is invalid or expired, your browser warns you -- do not ignore those warnings.

Password Security

Passwords are the most common authentication mechanism -- and the most commonly broken.

  • Password length matters most: a 20-character passphrase is stronger than a 10-character "complex" password with symbols
  • Use a different password for every account -- a breach at one site exposes all accounts if you reuse passwords
  • Password managers generate and store unique strong passwords for every site
  • Websites should store passwords as hashed + salted values, never as plaintext
  • Multi-factor authentication (MFA): something you know (password) + something you have (phone code) + something you are (biometric)

Network Security

  • Firewall: filters incoming and outgoing network traffic based on rules; blocks unauthorized connections
  • VPN (Virtual Private Network): encrypts traffic between your device and a VPN server; hides your IP address from sites you visit; useful on public Wi-Fi
  • TLS/SSL: the encryption protocol behind HTTPS; protects data in transit from interception
  • Two-factor authentication: protects accounts even if a password is compromised
  • Software updates: patches fix known vulnerabilities; delaying updates leaves known exploits open

The Digital Divide and Global Security

Security tools and practices are not equally accessible. This creates a digital divide in security:

  • Users in low-income regions may rely on outdated devices and software with unpatched vulnerabilities
  • Organizations in developing nations may lack resources for security training or infrastructure
  • Cybercrime is borderless -- attackers in one country can target anyone in the world
  • State-sponsored attacks (cyber espionage, infrastructure sabotage) are a major international security challenge
